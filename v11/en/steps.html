Using a FlowOrder node to control the order of execution paths in a flow

IBM Knowledge Center
Learn how to use the FlowOrder node to control the order of execution paths in a message flow driven by a JSON message over HTTP.
  • Create
  • Prepare
  • Run

Import projects

When you click Import, one project will be created in your workspace: An application named ExampleFlowOrder.

ExampleFlowOrder provides a single message flow named FlowOrder which provides two separate HTTPInput nodes as entry points to the flow. Both branches of the flow start by tracing the input with a Trace node and then use an HTTPReply to reply to the requestor.

The /FlowOrder branch of the flow then uses a FlowOrder node to execute two separate maps - the first one to lowercase the message data and the second one to uppercase the message data. Trace information is written to an output file.

The /RandomOrder branch of the flow does not control the order of execution of the two separate maps, so the user has no control over whether the message data is first converted to lowercase and then uppercase or vice versa. Trace information is written to an output file.

Once imported using the link on the right, the provided project is ready to be deployed and tested. To deploy, you can use a provided BAR file, as described on the Prepare tab.

Import

Prepare a trace directory, create an integration node and deploy the message flow

Prepare the scenario by executing the following steps:

  1. The message flow's Trace node will attempt to write a trace file to the directory C:\temp. Create this directory if it does not already exist, or edit the Trace node in the message flow to use a different directory suitable for your runtime environment.
  2. Next, create an integration node by starting an App Connect Enterprise command console and executing the following commands:
    mqsicreatebroker TESTNODE
    mqsistart TESTNODE
    mqsicreateexecutiongroup TESTNODE -e server
  3. Back in the Toolkit, drag and drop deploy the provided BAR file named FlowOrder.bar which you will find inside the ExampleFlowOrder application.

Use the Flow Exerciser to test the tutorial scenario

  1. Open FlowOrder.msgflow.
    • Click the Flow Exerciser icon to start testing the flow (when challenged you do not need to redeploy the flow again).
    • Click the Send Message icon .
    • Two saved messages are provided so you can execute both branches of the flow and compare the results. Each message provides the same input JSON message which contains mixed case data. To try out the FlowOrder branch select the input message FlowOrder. Try out the RandomOrder branch, select the input message RandomOrder. Click Send and your message is sent to the HTTPInput node.
    • Both FlowOrder and RandomOrder contain this data:
      {"message":"This is a MiXeD case message"}
    • After you close the dialog, the path taken through the messageflow is highlighted. Using a text editor, open the trace file C:\temp\TraceFlowOrder.txt. If you executed the FlowOrder branch, the trace file should read like this:
      2019-08-02 17:08:38.802437 = The FlowOrder path has been invoked ...
      2019-08-02 17:08:38.806927 = The message tree is set to: 'this is a mixed case message'
      2019-08-02 17:08:38.808863 = The message tree is set to: 'THIS IS A MIXED CASE MESSAGE'
      If you executed the RandomOrder branch, then the trace file entries may or may not be the other way around(!)
